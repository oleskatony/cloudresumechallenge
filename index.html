AWS Cloud Resume Challenge

This website was created as part of the Cloud Resume Challenge, an exciting initiative designed to highlight skills and explore various aspects of cloud computing. This project emphasizes the abilities required for a successful career in the cloud industry. For this endeavor, I focused on utilizing Amazon Web Services (AWS) as my preferred cloud provider. If you would like learn more about my attempt of the Cloud Resume Challenge, please check out my Github page that documents some of the components of this project.

This project has been an excellent learning experience that solidified my understanding of key cloud concepts. By applying Infrastructure as Code (IaC) principles with Terraform, implementing serverless solutions with DynamoDB and Lambda, and adopting CI/CD practices for website deployment, I have witnessed firsthand the efficiency and potential of AWS. I highly encourage others to take on this challenge as it not only helps in developing cloud skills but also serves as a remarkable platform to showcase their abilities.

3 Tier Architecture

To enhance my proficiency in AWS, I enthusiastically took on a project that involved provisioning resources for a 3-tier architecture. Multi-tier architecture is widely adopted in the industry, prioritizing security, high availability, and scalability by separating (decoupling) resources into 3 distinct layers or tiers.

This project served as a valuable complement to my prior involvement in the Cloud Resume Challenge, where my focus centered around serverless solutions. However, in this particular undertaking, I leveraged EC2 instances, multi-AZ RDS, and VPC components to design a solution akin to what one might encounter in a large enterprise environment.

In addition to designing and implementing the infrastructure for this project, I dedicated myself to creating a comprehensive breakdown of the entire undertaking. This involved thorough documentation, accompanied by supporting images and a diagram illustrating the architecture.
